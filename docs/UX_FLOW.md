# UX Flow

## UX 원칙

SmartClipboard는 “알아서 몰래 수집하는 앱”이 아니라 “사용자 액션을 최소화해 저장하고, 나중에 작업으로 연결하는 앱”입니다.

전체 경험은 ChatGPT/Codex처럼 단순하고 조용하게 구성합니다. 기능은 많지만 Home은 작업 시작과 작업 재개에 집중합니다. 포인트 컬러는 Galaxy/Samsung 느낌의 푸른색을 아주 절제해서 사용합니다.

## 톤앤매너

- 짧고 단정하게 말합니다.
- 기술 용어를 UI에 드러내지 않습니다.
- 사용자를 재촉하지 않습니다.
- 추천은 은근하게 유도합니다.
- 기본 저장 피드백: `SmartClipboard에 담았어요`
- 완료: `완료했어요`
- 전송: `캘린더로 보냈어요`, `노트로 보냈어요`
- 오류: `일부 항목을 처리하지 못했어요`, `다시 시도해 주세요`

## 탭 구조

### Home

- 새 작업 입력
- 이전 작업 리스트
- 이번 실행에서 새로 나온 AI 추천
- 자동 수집/분석 상태 카드
- 최근 자료 요약 접이식 패널

### Inbox

- 전체 자료 보관함
- 고정 카테고리 카드
- 자료 삭제, 다중 선택, 중요 표시, Topic 추가
- 그리드/리스트 전환

### Logs

- 사용자가 확인한 작업 중심 기록
- 사용자 요청, AI 추천 수락, Topic 생성, 초안 생성, 외부 앱 전송, 완료 상태 기록
- badge/status/source 필터

### Settings

- 자동 수집 범위
- 수집 기간
- 저장 용량 한도
- 자동 삭제 정책
- Gemini key 설정 안내
- 앱/권한 상태

## Home 화면 흐름

Home은 자료 목록이 아니라 작업 시작/작업 재개 중심입니다.

- 이전 작업은 심플한 리스트로 표시합니다.
- 사용자가 직접 질문해서 만든 작업도 남깁니다.
- AI가 자동 추천한 후보는 이번 실행/분석 사이클의 새 추천만 표시합니다.
- 과거 AI 추천 후보는 영구 보관하지 않습니다.
- 카드에는 작은 badge를 표시합니다.
  - `사용자 요청`
  - `AI 추천`
  - `진행 중`
  - `검토 필요`
  - `완료`

입력창 아래의 짧은 액션 칩:

- `최근 자료 정리`
- `새 추천 확인`
- `AI 다시 분석`
- `미완료 작업 보기`
- `수집 상태 보기`

도구 선택처럼 보이는 `요약하기`, `일정 만들기` 같은 칩은 기본 노출하지 않습니다.

## 링크/텍스트 저장 플로우

1. 사용자가 다른 앱에서 링크나 텍스트를 공유합니다.
2. Share Sheet에서 SmartClipboard를 선택합니다.
3. 투명/바텀시트 Activity가 잠깐 열립니다.
4. DataItem을 저장합니다.
5. 링크면 OG 추출을 1~2초만 시도합니다.
6. `SmartClipboard에 담았어요` 피드백을 보여주고 종료합니다.
7. 늦은 분석은 다음 앱 실행 또는 자동 분석 job에서 이어갑니다.

대체 흐름:

1. 사용자가 텍스트/링크를 복사합니다.
2. Quick Settings Tile을 누릅니다.
3. 투명 Activity가 focus를 얻고 Primary Clip을 읽습니다.
4. DataItem 저장 후 짧은 피드백을 보여줍니다.

## 스크린샷/이미지 저장 플로우

1. 사용자가 앱을 엽니다.
2. 앱은 Last Sync Time 이후의 모든 새 이미지를 MediaStore에서 조회합니다.
3. 새 이미지를 DataItem으로 자동 저장합니다.
4. OCR/이미지 분석을 자동으로 enqueue합니다.
5. 분석 상태는 Home의 작은 상태 카드에 표시합니다.
6. 사용자는 Inbox에서 삭제, 중요 표시, Topic 추가를 할 수 있습니다.

## 앱 실행 시 새 데이터 감지 및 정리 플로우

1. Home은 즉시 열립니다.
2. 자동 수집과 분석은 별도 상태로 진행합니다.
3. 상태 카드는 `새 자료 수집 중`, `OCR 분석 중`, `추천 준비 중`, `추천 준비됨`처럼 작게 표시합니다.
4. Gemini가 이번 실행의 Topic 후보를 생성합니다.
5. 새 추천은 Home에서 은근히 검토를 유도합니다.
6. 사용자가 추천을 열거나 직접 작업을 입력하면 Topic/Action 흐름으로 이동합니다.

## Inbox 플로우

Inbox 첫 화면은 갤러리 앱처럼 고정 카테고리 카드로 구성합니다.

카테고리 예:

- 이미지
- 스크린샷
- 다운로드 이미지
- 공유 링크
- 공유 텍스트
- 파일/PDF
- 중요 표시
- 미분석
- 최근 수집

각 카드를 누르면 해당 자료 모음으로 진입합니다. 자료 모음에서는 아주 작은 버튼으로 그리드/리스트 뷰를 전환합니다.

## Topic 생성 플로우

1. 사용자가 Home 입력창에 작업을 입력합니다.
2. AI가 관련 DataItem을 자동 선택합니다.
3. 사용자는 기본 선택을 그대로 진행하거나 직접 고르기로 전환합니다.
4. 선택 자료와 제목을 확정하면 Topic이 생성됩니다.
5. AI Agent가 초안을 생성합니다.

AI 추천에서 시작하는 경우:

1. 사용자가 AI 추천 카드를 엽니다.
2. AI가 만든 초안 중심 화면을 봅니다.
3. 사용된 자료는 `사용된 자료 12개`처럼 한 줄로만 표시합니다.
4. 필요하면 터치해서 자료 선택을 수정합니다.
5. 추천을 수락하면 Topic과 TopicAction이 저장됩니다.

## AI Agent 결과 확인 플로우

AI 결과 화면은 앱별 전송 카드 중심입니다.

- 상단: 전체 요약 짧게
- 카드: Samsung Notes, Samsung Calendar, Samsung Reminder 중 필요한 것만 기본 표시
- 각 카드 우상단: 작은 회색 `수정` 텍스트 또는 아이콘
- 카드 본문: 짧은 preview, 길면 `...`
- 카드 하단: 앱으로 보내기 버튼
- `다른 앱으로도 만들기` 또는 `추가`로 숨긴 카드 생성 가능

전송 후:

- 카드에 `전송됨` 배지와 체크 표시
- 카드 높이가 자연스럽게 줄어듭니다.
- 모든 필수 카드가 전송되면 작업이 자동 완료됩니다.
- 사용자는 언제든 즉시 완료할 수 있습니다.
- 미처리 액션이 있을 때 뒤로가기를 누르면 완료 처리/미완료 유지/취소를 선택합니다.

## Samsung Notes/Calendar/Reminder 검토 플로우

- Notes: 요약 노트 초안 확인 후 전송
- Calendar: 제목, 설명, 장소, 시작/종료 시간 확인 후 전송
- Reminder: 할 일 내용 확인 후 전송

Notes/Reminder는 삼성 앱 package를 지정합니다. Calendar는 삼성 캘린더를 우선하고, 없으면 기본 Calendar insert intent로 fallback합니다.

## 기존 UI에서 참고할 것

- 투명 Activity 피드백 패턴
- One UI inspired blue palette
- DataItem 타입 필터
- 선택 상태 유지 아이디어
- Handoff formatter/launcher 초기 구조

## 기존 UI에서 버릴 것

- 데이터 리스트가 앱의 중심처럼 보이는 흐름
- 휴리스틱 추천을 AI처럼 보이게 하는 표현
- 대형 단일 `MainScreen.kt` 중심 구조
- 자동 수집 자료를 홈에 너무 크게 펼치는 구성
- 사용자가 다음에 무엇을 해야 하는지 모호한 CTA
